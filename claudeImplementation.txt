│ HospitalTriaje — Implementation Plan                                         │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ A cross-platform mobile + web triage application that guides patients        │
│ through a self-evaluation                                                    │
│ (Manchester Triage System), recommends the best nearby hospital based on     │
│ location, specialist                                                         │
│ availability and real-time wait times, and delivers emergency action         │
│ recommendations for critical                                                 │
│ cases. Hospital staff update their facility status via API tokens. The       │
│ system must be usable                                                        │
│ partially offline and push results to patients via FCM/APNs.                 │
│                                                                              │
│ ---                                                                          │
│ Confirmed Requirements                                                       │
│                                                                              │
│ Dimension: Frontend                                                          │

│ ---                                                                          │
│ MTS Triage Engine                                                            │
│                                                                              │
│ - Question tree stored as directed graph in seed/question_tree.json          │
│ - Each node: { id, question_es, options: [{ label_es, next_node_id }] }      │
│ - Leaf nodes: { triage_level: 1–5, complaint_category, max_wait_minutes }    │
│ - triage_engine.py traverses the graph given a list of (node_id,             │
│ answer_index) pairs                                                          │
│ - Output: { level, label, max_wait_minutes, complaint_category }             │
│                                                                              │
│ Hospital Routing Algorithm                                                   │
│                                                                              │
│ score = (distance_km × 0.4) + (wait_time_min × 0.4) - (specialist_match × 10 │
│  × 0.2)                                                                      │
│ - Exclude hospitals where required specialist is unavailable at current time │
│  (schedule + override)                                                       │
│ - Sort ascending by score; return top 5                                      │
│                                                                              │
│ SSE Data Flow                                                                │
│                                                                              │
│ Staff POST /hospitals/{id}/status                                            │
│   → hospital_routing.update_status() persists to DB                          │
│   → sse_manager.broadcast({ hospital_id, wait_time_min, available_beds,      │
│ updated_at })                                                                │
│   → All subscribed SSE generators yield the event                            │
│   → Flutter SSE client updates Riverpod state                                │
│   → Map + list UI rebuilds                                                   │
│                                                                              │
